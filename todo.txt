# DEVOPS TODO LIST — HTTPS / SSL SETUP & FRONTEND CONNECTION

Use this guide tomorrow to complete the frontend connection and bypass the HTTPS "Mixed Content" security blocks.

## TASK 1: CHOOSE YOUR ROUTE

Select either **Route A** (deploying fully on HTTPS via Vercel) or **Route B** (local testing on your computer).

### ROUTE A: Setup Free SSL on EC2 via DuckDNS (Recommended for production/sharing)
Since browsers block HTTPS -> HTTP calls, you need to secure your EC2 backend (`47.129.27.231`) with a domain name and SSL.

1. **Register a free subdomain**:
   * Go to [duckdns.org](https://www.duckdns.org/) and log in (GitHub, Google, etc.).
   * Under **subdomains**, create a name (e.g., `finance-flow-api-zen`).
   * In the **IP** address field, enter: `47.129.27.231` and click **update ip**.
   * *Domain is now: `<subdomain>.duckdns.org`*

2. **Configure SSL on EC2 server**:
   * SSH into your server (or open EC2 Instance Connect browser terminal).
   * Run these commands:
     ```bash
     # Install Certbot for Nginx
     sudo apt-get install certbot python3-certbot-nginx -y

     # Generate Certificate (replace with your chosen DuckDNS subdomain)
     sudo certbot --nginx -d <subdomain>.duckdns.org
     ```
   * *Follow prompts (enter email, agree to terms). Nginx will automatically reload with active SSL.*

3. **Update Vercel Environment Variables**:
   * Go to your Vercel Dashboard for the frontend project.
   * Under **Settings > Environment Variables**, update `NEXT_PUBLIC_API_URL` to:
     ```text
     https://<subdomain>.duckdns.org
     ```
   * Re-deploy the frontend on Vercel to load the new environment variable.

---

### ROUTE B: Run Frontend Locally (Quickest local test)
If you do not want to configure a domain name, you can run the frontend on `localhost` (which is HTTP and can communicate with your HTTP EC2 backend without mixed content blocks).

1. Open your terminal on your MacBook.
2. Navigate to the `frontend` folder:
   ```bash
   cd frontend
   ```
3. Open/create `frontend/.env.local` and set:
   ```text
   NEXT_PUBLIC_API_URL=http://47.129.27.231
   ```
4. Install dependencies and start the app:
   ```bash
   npm install
   npm run dev
   ```
5. Open `http://localhost:3000` in your browser. It will connect to the remote EC2 server successfully.
